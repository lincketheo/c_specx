# Contributing to NumStore

Contributions are welcome as of January 2026, with review. Reach out directly if you're interested.

## Talk to me first

This is a small project and I'd like to stay in the loop before work begins. Open an issue or reach out before you start coding — it avoids wasted effort and keeps things moving in a consistent direction. All pull requests go through my review before merging.

## AI usage

Using AI to understand the codebase, brainstorm approaches, or work through problems is fine. Submitting PRs that are clearly wholesale AI-generated without real understanding or review on your end is not. If I can tell the code wasn't written or vetted by a human who understands what it does, it'll be rejected.

## Getting started

1. Fork the repo and clone it locally
2. Read the [Quick Start](quickstart.md) and [C API](library-api.md) docs
3. Open an issue describing what you want to work on
4. Wait for a thumbs up before writing code
5. Submit a PR with a clear description of what changed and why
